1. ROS2 Context

ROS2 Context 是一个用于配置和管理 ROS2 节点上下文的工具，其主要作用是设置运行环境，使得 ROS2 节点可以运行在指定的域内。

1.1 输入参数

ROS2 Context 的输入参数包括以下内容：

1.1.1 domain_id

作用： 指定 ROS2 节点运行的域 ID。
说明： 域 ID 是 ROS2 中用于划分节点运行区域的标识，只有运行在同一域内的节点才能进行通信。
典型值： 根据需求设置，常见为 0 或特定的网络配置值。

1.1.2 useDomainIDEnvVar

作用： 决定是否使用环境变量中的域 ID。
说明： 如果设置为 True，则会优先使用环境变量中的 domain_id 值，确保与其他实例一致。

1.2 输出参数

ROS2 Context 的输出参数是：

context: 分配给 ROS2 上下文的唯一标识符。

2. 踩过的坑

  • 未正确配置 domain_id 导致节点通信失败。
  • 未勾选 useDomainIDEnvVar 导致参数冲突。
